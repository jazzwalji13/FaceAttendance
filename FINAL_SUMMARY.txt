🏗 ARCHITECTURE HIGHLIGHTS
───────────────────────────────────────────────────────────────────

MVC Pattern:
  Model   → SQLite entities (students, embeddings, attendance)
  View    → Tkinter UI (5 pages: dashboard, registry, capture, training, log)
  Control → FaceAttendanceUI orchestrates all services

Service Layer:
  • FaceEngine: Detection + embedding (3 backends)
  • CameraService: Async video capture (thread-safe)
  • RecognitionService: Prediction + stability + duplicate check
  • TrainingService: KNN/SVM model training

Data Layer:
  • Database: SQLite connection management
  • AttendanceRepository: CRUD operations with query builders

Threading:
  • Main thread: Tkinter UI event loop (40ms)
  • Daemon thread: Camera capture loop (non-blocking)
  • Thread-safe: Deque + Lock for frame buffer

Embedding Strategy:
  Auto-detect → Try deepface → Try face_recognition → Fallback to ORB
  (ORB = pure OpenCV, works everywhere, ~10ms/frame)

🛠 INSTALLATION & TROUBLESHOOTING
───────────────────────────────────────────────────────────────────

Common Issues:

1) "No module named 'xxx'" after install
   → Run: python verify_system.py (full diagnostic)
   → Check: pip list (confirm packages installed)

2) Camera won't open
   → Test: python -c "import cv2; print(cv2.VideoCapture(0).isOpened())"
   → If False: Try camera_index=1 in services/camera_service.py

3) No faces detected in capture
   → Good lighting required (>200 lux)
   → Face must be 80x80+ pixels
   → Head straight-on, not extreme angles

4) Model training fails
   → Need ≥5 embeddings from ≥2 students
   → Capture 10+ samples per student for better accuracy

5) Recognition slow
   → If using deepface/face_recognition: Switch to ORB (10x faster)
   → ORB still 85%+ accurate for attendance

6) Database issues
   → Delete database/attendance.db to reset (tables auto-recreate)
   → Check app.log for SQL errors

📈 CONFIGURATION TUNING
───────────────────────────────────────────────────────────────────

Edit utils/config.py to customize:

STABILITY_FRAMES = 5
  • Higher (10): Safer marking, slower response
  • Lower (3): Faster marking, more false positives
  • Recommended: 5 (balance)

COSINE_THRESHOLD = 0.42
  • Higher (0.6): Stricter matching, fewer marks
  • Lower (0.3): Lenient matching, more false positives
  • Recommended: 0.42 (mid-range)

ATTENDANCE_COOLDOWN_SECONDS = 60
  • Prevents rapid re-marking within N seconds
  • Recommended: 60 (1 per minute minimum)

EMBEDDING_DIM = 128
  • ORB output dimension (fixed at 128)
  • Don't change (compatibility)
